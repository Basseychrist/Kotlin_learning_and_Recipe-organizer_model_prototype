# Overview

As a software engineer, I am building a strong foundation in programming languages to advance my technical skills and expand my ability to solve complex problems. Through this project, I aimed to deepen my understanding of Kotlin's core features—functions, control flow, loops, collections, and object-oriented design patterns. By mastering a modern JVM-based language, I'm broadening my toolkit and preparing myself for real-world software development.

I developed a comprehensive Kotlin learning project consisting of two main components: a series of focused learning modules that explore fundamental language concepts (functions, conditionals, loops, collections, and class mechanics), and a Recipe Organizer & Smart Meal Planner application that demonstrates variable scope, collection manipulation, and data structure design in a practical context. The learning modules provide isolated examples for studying specific concepts, while the Recipe Organizer showcases how these concepts integrate to build a functional application.

The purpose of this software is to create a structured learning resource that reinforces Kotlin syntax and language fundamentals through both isolated examples and integrated applications. By combining theory with practice, I'm building confidence in using Kotlin and demonstrating my ability to apply language features to real-world problems like recipe management and meal planning.

# Development Environment

I used **IntelliJ IDEA Community Edition** as my primary development environment. IntelliJ provides comprehensive Kotlin support with syntax highlighting, code completion, inline documentation, and integrated compilation tools. The IDE streamlined development and helped me understand Kotlin syntax through real-time feedback and intelligent code suggestions.

This project is written entirely in **Kotlin** (version 1.8+), a modern JVM language that combines object-oriented and functional programming paradigms. The software runs on the **Java Virtual Machine (JVM)** (version 11+) and uses **Gradle** as the build tool. The project has **no external dependencies**—it relies solely on Kotlin's standard library for core features like collections, functions, and control flow. This approach keeps the project lightweight and focuses attention on language fundamentals rather than framework-specific patterns.
